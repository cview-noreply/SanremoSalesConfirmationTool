# サンレモ成約捕捉 ツール — exe 化・配布手順書

---

## 概要

本手順書では、Pythonがインストールされていない共有PC・配布先PCでも  
`サンレモ成約捕捉.exe` をダブルクリックするだけで起動できるようにする手順を説明します。

**ビルドはPythonが入っている開発PC（1台）で実施するだけです。**  
配布先にはPythonのインストールは不要です。

---

## ファイル構成

以下のファイルを同じフォルダにまとめてください。

```
作業フォルダ/
│
├── app.py                      # メインスクリプト
├── create_sheets.py
├── check_sheets.py
├── create_sendlist.py
├── store_documents.py
├── create_alert.py
├── utils.py                    
│
├── config.yml                  # 設定ファイル
│
├── サンレモ成約捕捉.spec        # ビルド設定
└── build.bat                   # ビルド実行バッチ
```

> `config.yml` を実行ファイルと同じフォルダから読み込むように修正されています。

---

## ビルド手順（開発PC側）

### Step 1: 作業フォルダの準備

上記のファイルをすべて同じフォルダに配置します。

### Step 2: `build.bat` を実行

`build.bat` をダブルクリックします。  
初回は必要ライブラリのインストールが自動で行われるため、数分かかります。

```
ビルド完了後の出力先:
  dist\サンレモ成約捕捉\
```

### Step 3: config.yml を dist フォルダにコピー

ビルド完了後、`dist\サンレモ成約捕捉\` フォルダにconfig.ymlを手動でコピーしてください。

| ファイル名 | 説明 |
|-----------|------|
| `config.yml` | 環境に合わせて記載内容を編集する |

## 配布先フォルダ構成（最終形）

配布先PCには、以下の構成のフォルダごと共有・コピーしてください。

```
サンレモ成約捕捉\                    ← このフォルダごと配布
│
├── サンレモ成約捕捉.exe              ← ダブルクリックで起動
├── config.yml                       ← ★配布先のパスに書き換える
├── _internal\                       ← 自動生成。削除不可
└── （その他DLLなど）
```

---

## 配布先での config.yml 編集

配布先PCでは、`config.yml` を開き **パスを各PC向けに書き換えてください。**

```yaml
# 例: 配布先PCのユーザー名が UU123456 の場合
ルートフォルダ: 'C:\Users\UU123456\Documents\.SANREMO\自動化\全国化'

ファイルパス:
  PW一覧: 'C:\Users\UU123456\Documents\.SANREMO\...\クライアント別PW一覧.xlsx'
  # 以下同様に書き換え
```

> **共有フォルダ運用の場合:** ルートフォルダをネットワークドライブのパス  
> （例: `\\server\share\SANREMO\全国化`）に変更することで、  
> 複数人が同じ設定で使用できます。

## よくあるエラーと対処法

| エラー | 原因 | 対処 |
|--------|------|------|
| `ReferencePathError: 以下の設定パスが見つかりません` | `config.yml` のパスが間違っている | `config.yml` を開いてパスを確認・修正 |
| 起動直後に画面が出ない | pywebview の初期化に時間がかかっている | 最大10分ほど待つ |
| `xlwings` エラー | Excelがインストールされていない | 配布先PCにExcelが必要（xlwingsはExcelに依存） |
| ビルド時 `ModuleNotFoundError` | ライブラリが不足 | `pip install <ライブラリ名>` で追加後、再ビルド |

---

## 注意事項

- **配布先PCにもMicrosoft Excelのインストールが必要です。**  
  xlwingsはExcelのCOMオブジェクトを使用するため、Excelなしでは動作しません。
- `_internal\` フォルダは削除・移動しないでください（exeが起動しなくなります）。
- セキュリティソフトがexeの実行をブロックする場合は、除外設定を追加してください。
- ビルドはPythonのバージョン依存があります。**Python 3.10〜3.12** の使用を推奨します。
